# puzzle-nonograms-solver

一个专门用于解决 [cn.puzzle-nonograms.com](https://cn.puzzle-nonograms.com) 上数织（Nonograms）谜题的求解器。

## 项目简介

本项目旨在自动求解数织（Nonograms）谜题。数织是一种经典的逻辑绘图游戏，玩家需要根据网格左侧和顶部的数字提示，推断出哪些格子需要填充，最终拼出一幅隐藏的像素画。

该求解器由两部分组成：
- **C++ 核心求解算法** (`solve.cpp`)：实现动态规划谜题求解。
- **Python 数据处理脚本** (`data.py`)：负责从网站获取谜题数据。

## 功能特点

- 针对 [cn.puzzle-nonograms.com](https://cn.puzzle-nonograms.com) 的谜题格式进行优化。
- 适合学习数织求解算法（如回溯法、行列推理等）的参考实现。

## 文件说明

| 文件 | 语言 | 说明 |
|------|------|------|
| `solve.cpp` | C++ | 核心求解器，实现数织谜题的主要逻辑。 |
| `data.py` | Python | 辅助脚本，用于爬取、解析谜题数据。 |

## 算法分析

这个算法是一个典型的**约束传播 + 回溯搜索**的非ogram求解器。它的核心思想是：利用每一行/列已知的数字提示，尽可能推导出每个格子是必须涂黑还是必须留空；当推理无法继续时，再通过试探（猜测）某个格子的状态，并递归求解，直到找到完整解或发现矛盾。

下面分层次讲解它的原理。

---

### 1. 问题与挑战

Nonogram（绘图逻辑）是一个二进制网格填色谜题：
- 每行左边给出该行黑色连续块的长度（如 `4` 表示一整段长度为4的黑色块；`1 1 3` 表示三段黑色块，长度分别为1、1、3，它们之间至少有一个空格隔开）。
- 每列上方也有类似的数字提示。
- 目标是确定每个格子是黑（`x`）还是白（`o`）。

约束条件很多（行和列必须同时满足），直接暴力搜索所有2^(m×n)种状态是不可能的，所以需要用推理大幅减少搜索空间。

---

### 2. 总体框架：传播 + 回溯

```
求解过程：
1. 反复对每一行、每一列调用“单行求解”，推导出必须黑/必须白的格子。
2. 如果某行/列出现矛盾（无合法方案），则整个问题无解。
3. 当推理不再产生新信息时，如果网格还有未确定的格子，就选择一个格子进行猜测（假设它是黑，尝试求解；若失败则假设它是白，再尝试求解）。
4. 递归进行，直到所有格子确定或证明无解。
```

这个框架与很多精确覆盖问题的回溯算法类似，但关键在于“单行求解”的推理非常高效，能在大多数情况下直接确定大部分格子，使得回溯深度很小。

---

### 3. 单行/列求解（DP 推理）

给定一行当前的状态（有些格子已确定黑/白，其余未知）和该行的数字提示，我们要找出**所有可能的合法填法**的**交集**，从而得出那些在所有合法方案中状态一致的格子。

#### 状态定义

把行看作一个长度为 `n` 的序列，数字提示为 `clues[0..k-1]`（k 个色块）。  
定义状态 `(j, i)` 表示：已经处理完前 `j` 个色块，下一个要决定的位置是 `i`（即位置 `0..i-1` 已经合法放置了前 `j` 个色块）。

#### 转移

从状态 `(j, i)` 可以采取两种动作：

1. **跳过位置 `i`（留空）**：如果当前位置 `i` 没有被强制为黑（即 `board[i] != 'x'`），则转移到 `(j, i+1)`。
2. **放置第 `j` 个色块**：色块长度为 `c = clues[j]`，占据区间 `[i, i+c)`。要求：
   - 区间内所有格子不能已经被标记为白（`canFill` 检查）。
   - 如果色块恰好到行尾（`i+c == n`），则转移到 `(j+1, n)`。
   - 否则，色块后必须紧跟一个空格，所以要求位置 `i+c` 没有被强制为黑，转移到 `(j+1, i+c+1)`。

#### 前向 DP（可达性）

从起点 `(0,0)` 开始，按上述转移规则标记所有能到达的状态。  
`fwd[j][i] = true` 表示从起点可以到达状态 `(j,i)`。

#### 后向 DP（反向可达性）

从终点 `(k,n)` 开始，反向执行转移，标记所有能到达终点的状态。  
`bwd[j][i] = true` 表示从状态 `(j,i)` 可以到达终点。

#### 确定格子的状态

对于位置 `i`，考虑所有可能的转移：

- **跳过转移**：如果存在某个 `j` 使得 `fwd[j][i] && bwd[j][i+1]` 成立，则意味着某个合法方案中位置 `i` 是留空的。  
- **放置转移**：如果存在某个 `j` 使得 `fwd[j][i] && bwd[j+1][e]` 成立（其中 `e` 是放置色块后的下一位置），则意味着某个合法方案中位置 `i` 是涂黑的，且色块覆盖了 `[i, e)`。

利用差分数组统计每个格子被“可以涂黑”的转移覆盖的次数，以及被“可以留空”的转移覆盖的次数。  
最终：
- 如果某个格子只能涂黑（`can_filled` 真且 `can_empty` 假），则标记为黑。
- 如果只能留空（`can_empty` 真且 `can_filled` 假），则标记为白。
- 如果两者都真，则状态仍不确定，保持未知。
- 如果两者都假，则存在矛盾（无合法方案）。

这样，我们就能把一行中所有可能方案的交集提取出来，更新格子。

---

### 4. 传播（Propagation）

对每一行执行一次单行求解，可能会改变一些格子；然后对每一列也执行一次单行求解。  
由于行和列的求解相互依赖（行更新后可能使列获得新信息，列更新后又可能影响行），因此需要反复进行，直到某一轮没有任何格子被更新（到达不动点）。  
这个循环确保了当前所有已知的行列约束都被充分利用，最大程度地减少了未知格子。

---

### 5. 回溯（Backtracking）

当传播结束后，如果仍有格子是未知的，就选择一个未知格子进行猜测。  
通常选择能最快产生矛盾的格子（比如可选方案最少的），但代码中简单选择了第一个遇到的未知格子。  
然后递归求解：

- 先假设该格子为黑（`'x'`），调用 `solve`。
- 如果成功，返回真。
- 否则恢复棋盘，假设该格子为白（`'o'`），再次调用 `solve`。
- 如果两种都失败，则当前分支无解，返回假。

这种深度优先搜索加上约束传播，使得回溯很少发生，因为传播阶段已经解决了大部分格子。

---

### 6. 复杂度与优化

- **单行求解**：DP 的时间复杂度为 `O(n·k)`，其中 `n` 是行/列长度，`k` 是提示数。
- **一轮传播**：对 R 行和 C 列各做一次，复杂度 `O((R+C)·n·k)`。
- **传播轮数**：最坏情况下可能达到 `O(n)` 轮，但实际通常很快收敛。
- **回溯**：最坏是指数级，但由于推理强大，实际求解大型谜题也很快。

该算法使用差分数组来高效统计可涂黑位置，避免了 O(n²) 的重复计算，是一种经典的 Nonogram 求解实现。

---

### 总结

这个算法通过**前向-后向 DP** 精确刻画了每一行/列的所有合法填充方案，并取其交集来强制确定格子。然后利用**约束传播**让行列信息相互加强，最后在必要时使用**回溯搜索**来完成求解。这种结合方式使得算法在大多数非ogram谜题上都能高效求解，是竞赛和实际应用中常见的解法。
